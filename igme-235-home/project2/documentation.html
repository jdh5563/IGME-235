Design Process

  • API Project Goal

    My goal with this project is to create an application that can effectively utilize the information available to it in the Dog API to provide users with a fun experience to look at dogs, or even with something practical like providing help to decide what breed of dog they should look into getting.

  • Audience

    The audience for my web app is anyone interested in dogs, whether it's because they want a new pet, are looking for a cute wallpaper, or even because they just want to see cute pictures of dogs.

  • How Is My Content Organized?

    At the top of the page, there is an image of puppies with the title "Dog Heaven" laid on top of it. Underneath is a search box with drop-down menus to select the breed of dog to look at and the sub- breed if there is one, followed by a drop-down menu for how many images to load and a button to begin searching for images. Beneath that is a section for all of the resulting images to pop up in a grid format, with a button above each image to view an enlarged version of the image in a new tab. If there are multiple types of dogs in the search results, then the breed and subbreed are displayed over each image. If there is only one type of dog, then there is no heading above the images.

How I Met the Requirements

  • My app saves information from all changeable search boxes.

  • My app has 2 search fields, a menu to change how many images are retrieved, and buttons to enlarge images.

  • My app has custom HTML and CSS, all widgets are labeled, there is enough information on the page for users to know what they're doing, any errors are managed appropriately, the state of the app is always apparent, the app is presentable on a fullscreen format and on mobile, I use CSS Flexbox and Grid, my html and css validates, my JavaScript is stored in a separate file and is well documented.
